1Get UniProt IDs for a named set

When accessions are all you need, resolve() returns a plain list. resolve_dynamic() additionally hands back a {accession: sequence} dict and the full DataFrame.

code
from protein_sets import resolve, resolve_dynamic

ids = resolve("secretome")          # → list of UniProt accessions
len(ids), ids[:8]
output
(2421, ['A0A0K2S4Q6', 'A1E959', 'A1KZ92', 'A1L4H1', 'A2VEC9', 'A4D0S4', 'A4D1T9', 'A5D8T8'])
code
ids, seqs, df = resolve_dynamic("secretome")
ids[0], len(seqs[ids[0]]), seqs[ids[0]][:30] + "…"
output
('A0A0K2S4Q6', 201, 'MTQRAGAAMLPSALLLLCVPGCLTVSGPST…')

2Query directly — the proteome and the named sets

Load the whole reviewed-human table, then resolve any registered set to a live count. Numbers are computed from the cached data, never hardcoded.

len(prot), prot.shape[1]            # rows, columns

4Get sub-sequences — domains of signal-peptide proteins

The headline move: slice regions within sequences. Each region row carries start/end coordinates (1-indexed, inclusive) and the exact sliced sequence. Here we take everything in the signal_peptide set and pull out its domains two ways.

6Transform teaser — a mutation scan

Transforms produce synthetic sequences from real proteins, on the same row contract. mutation_scan walks a sliding window across the sequence, substituting each position with a panel of amino acids.

code
from protein_sets import mutation_scan

# insulin: 110 residues × 4-AA alphabet (A,R,E,F) → 440 mutants + 1 WT
df = mutation_scan("P01308")
len(df), df["item_id"].head(6).tolist()
output
(441, ['P01308_wt', 'P01308_pos0000_A', 'P01308_pos0000_R', 'P01308_pos0000_E', 'P01308_pos0000_F', 'P01308_pos0001_A'])

The item_id scheme is self-describing: {uid}_wt for the wild type and {uid}_pos{NNNN}_{aa} for each mutant, with the position zero-padded so lexicographic sort equals numeric sort. The frame also carries kind, pos, aa, win_start, win_end, win_len — and feeds esm_embed.embed_regions(df, …) exactly like a real-protein region set.
